# Company settings schema (kiho v6)

**Status:** foundation shipped in v6.0.0-alpha.1 (PR #1).
**Full propagation wired:** v6.0.0-beta.1 (PR #2) and v6.0.0 (PR #3).

The company settings file lives at `$COMPANY_ROOT/settings.md` (YAML frontmatter + markdown body). It is read by the CEO at `INITIALIZE` step 1, immediately after the plugin-level `config.toml`, with company values **overriding** plugin defaults key by key.

## Scaffolding

`kiho-setup` auto-scaffolds the file on first v6 turn if missing. Users never need to create it by hand. To regenerate after accidental deletion:

```
/kiho:kiho kb-init   # re-runs kiho-setup which notices settings.md is missing
```

## Precedence

For any given key, the value used at runtime is the first non-null hit in this order:

1. `$COMPANY_ROOT/settings.md` frontmatter
2. `plugins/kiho/skills/core/harness/kiho/config.toml`
3. hardcoded plugin default

Runtime merge happens in-memory at CEO INITIALIZE; no file-level rewrite of config.toml occurs.

## Full frontmatter schema

```yaml
---
schema_version: 1

company_metadata:
  name: string                   # display name for this company
  primary_timezone: string       # IANA tz (e.g. "Asia/Hong_Kong")

official_language: string        # BCP 47 (en, zh-TW, zh-CN, ja, ko, ...)
                                 # Propagates to:
                                 #   - every subagent brief (appends "Output language: <value>")
                                 #   - committee transcripts (opened in this language)
                                 #   - CEO narration to user (unless user writes in a different lang)

tone:
  formality: string              # "collaborative-formal" | "terse" | "playful"
  emoji_in_agent_output: bool    # governs orchestration output; NOT UI output of built apps

recruit:
  auto_trigger_on_gap: bool
  min_candidates_always: int     # minimum 4 per v6 plan
  committee_gate_threshold: float      # 0.0..5.0 rubric
  synthesis_when_complementary: bool
  synthesis_rubric_delta_max: float
  memory_seed_on_hire: bool
  max_auto_recruits_per_turn: int
  skill_research_before_design: bool
  max_skills_authored_per_recruit: int

skill_library:
  auto_consolidate_research: bool
  reuse_threshold_count: int
  consolidate_cadence_days: int
  consolidate_cadence_new_skill_count: int
  stale_days: int

kb_consolidation:
  project_kb_cadence_turns: int
  company_kb_cadence_days: int
  company_kb_cadence_turns: int

startup:
  read_company_wiki_on_init: bool
  read_settings_on_init: bool
  warn_if_stale_days: int
  warn_if_agent_mtime_older_than_project: bool

promote:
  auto_scope_classify_on_done: bool
  dry_run_before_write: bool

performance:
  rank_skills_by_performance: bool
  rank_weights:
    success_rate: float           # must sum to 1.0 with others
    inverse_correction_rate: float
    freshness: float

external_skills:
  allow_references: bool
  catalog_ttl_days: int
---
```

## Key interactions

### `official_language`

When non-null, appended to every subagent brief under `Company output constraints`. Also:

- `committee` skill reads this before opening a transcript; transcript markdown is authored in that language
- CEO narration (ledger entries, user summary, AskUserQuestion prompts) is localized to this language
- Agent personas generated during hire default to bilingual work-sample text (primary = this lang, fallback = en)

Fallback chain: `settings.official_language` → `config.toml.tone.official_language_fallback` → `"en"`.

### `recruit.*`

Consumed by the universal gap-healing reflex in CEO INITIALIZE step 11 and LOOP steps b/c:

- `auto_trigger_on_gap = false` reverts to v5 behavior (ASK_USER on RACI fail)
- `min_candidates_always` is a HARD FLOOR — cannot go below 4
- `committee_gate_threshold` bypassable only by unanimous auditor vote
- `max_auto_recruits_per_turn` prevents runaway; overflow triggers user prompt

See `skills/core/hr/recruit/SKILL.md` and `references/agent-schema-v2.md`.

### `skill_library.*` and `kb_consolidation.*`

Consumed by the consolidation reflex at CEO DONE step 10b:

- Cadence gates: skill-library runs if `days_since_last >= consolidate_cadence_days` OR `new_skills_since_last >= consolidate_cadence_new_skill_count`
- Similar for project-KB (turn-based) and company-KB (days OR turns)
- `stale_days` controls skill-library's deprecation pass

### `startup.read_company_wiki_on_init`

When true, CEO INITIALIZE step 1 reads (in this order):

1. `$COMPANY_ROOT/settings.md` — merges into runtime config
2. `$COMPANY_ROOT/company/wiki/index.md` — company KB index snapshot
3. `$COMPANY_ROOT/skills/INDEX.md` — skill library index snapshot

All three reads are tolerant of missing files (warn + continue); the files are auto-scaffolded/generated by subsequent CEO DONE steps.

### `promote.auto_scope_classify_on_done`

When true, CEO DONE step 3 invokes `scope-promote-classifier` against files created in `.kiho/audit/**` and `.kiho/state/research/**` during this turn. Classifier output:

- `project` — leave in project
- `company` — copy via `kb-manager.kb-add` to `$COMPANY_ROOT/company/wiki/<category>/`
- `split` — extract `§generic` sections into company; leave rest

`promote.dry_run_before_write = true` (default) presents the diff via `AskUserQuestion` before any company-tier write.

### `performance.*`

Consumed by design-agent Phase 2.3 and Phase 3.5.5 when multiple existing skills could cover a need:

```
score(skill) = (w_s × success_rate)
             + (w_c × (1 − correction_rate))
             + (w_f × freshness)
```

where `freshness = max(0, 1 − days_since_last_invocation / 90)` and `w_s + w_c + w_f` must equal 1.0.

Skills ranked top go to USE; middle → IMPROVE candidate; bottom (`score < 0.4` AND not referenced) → deprecate candidate for next consolidation cycle.

### `external_skills.*`

Consumed by `skill-discover` (PR #3) when scanning `$CLAUDE_PLUGINS` for referenceable external skills. When `allow_references = true`, a skill's `references:` block may point to:

- `internal_skill` — another `$COMPANY_ROOT/skills/<id>` entry
- `plugin_skill` — e.g. `onchainos:okx-dex-token`
- `claude_global_skill` — e.g. `firecrawl:firecrawl`
- `external_docs` — URL with purpose note

`catalog_ttl_days` determines how often the cache at `$COMPANY_ROOT/external-skills-catalog.json` is refreshed.

## Lint

Basic schema validation happens at CEO INITIALIZE step 1 (YAML parse + required-key check). Deeper validation (rank_weights sum to 1.0, enum values, integer ranges) happens via a future `bin/company_settings_lint.py`.

## Migration

No migration needed — the file simply didn't exist in v5. On first v6 turn, kiho-setup scaffolds the default and proceeds. User may edit immediately afterward.
