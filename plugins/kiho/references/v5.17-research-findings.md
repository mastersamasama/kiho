## Headline findings

### H1. The taxonomy collapses to 6 factory patterns

Six recurring shapes of "AI artifact factory" emerged from 2024-2026 industry + academic literature. Every viable factory is a composition of these six. None is dominant on its own.

### H2. Anthropic's `skill-creator` already implements much of what kiho lacks

`skills/skill-creator/scripts/run_loop.py --max-iterations 5` plus 20 trigger-eval queries with 60/40 train/test split + best-by-test-score selection (prevents overfitting) + dedicated `agents/grader.md`, `agents/comparator.md`, `agents/analyzer.md` subagents. This is the single closest reference architecture for the kiho `_meta` factory.

### H3. GEPA (ICLR 2026 Oral) is the strongest 2026 primary source

GEPA maintains a **Pareto frontier of prompt candidates** rather than evolving one global best. Outperforms GRPO by 6-20% with up to 35× fewer rollouts. Directly applicable to kiho's "single-writer-pattern parity across siblings" gap — instead of forcing one canonical pattern, surface the frontier and choose per-sibling.

### H4. Karpathy autoresearch is the canonical minimal closed loop

Agent edits one fenced file (`train.py`), runs it for fixed 5-min budget, scores `val_bpb`, **keeps if improved else discards**. Files like `prepare.py` are explicitly fenced "do not modify". Ran 700 experiments / 20 wins overnight. The pattern requires (a) a scalar score, (b) a fenced editable surface, (c) a fixed iteration cost. kiho currently has none of these explicitly per-skill.

### H5. Toyota/Shingo distinction is the exact diagnosis for kiho

> **Shingo Institute on mistake-proofing:** *"Mistakes are inevitable, but defects (mistakes that reach the customer) are preventable through poka-yoke."*

Every gap on the user's observed-defect list is a *mistake* (paraphrased citation, wrong script location, missing smoke test) that became a *defect* (reached the user for review) because no poka-yoke was installed in the pipeline. Deming sharpens it: *"Inspection is fundamentally flawed as a quality control method and is merely a reactive measure."* kiho's 24 gates are inspection. The factory needs to install the poka-yokes upstream so the mistakes never become defects.

## Factory architecture taxonomy — 6 patterns

### Pattern 1 — Compile-via-Optimizer (declarative → tuned prompt)

Author writes a **program signature + a metric**, never a final prompt. An optimizer searches instruction + few-shot space until the metric clears. DSPy's `BootstrapFewShot` filters teacher-generated demos through the metric. **MIPROv2** runs three stages: bootstrap traces → grounded instruction proposal → Bayesian discrete search. **GEPA** replaces RL with reflective natural-language critique on trajectories. Convergence signal: metric clearing on a **held-out test split**.

Sources: dspy.ai/learn/optimization/optimizers/, arXiv 2507.19457.

### Pattern 2 — Scaffold-and-Fill (template + parameter schema)

Backstage Software Templates is the canonical non-AI version. Author defines a **JSONSchema for parameters** and an **ordered `spec.steps[]` array**; each step has an `if:` directive and a typed `input:`. A **review page** renders parameters before any step executes; **dry-run mode** previews the full filesystem result.

Sources: backstage.io/docs/features/software-templates/writing-templates, writing-custom-field-extensions.

### Pattern 3 — Generate-then-Verify-Loop (propose → score → keep/discard)

Karpathy's `autoresearch` (Mar 2026, 21k stars) is the minimal canonical form. The pattern *requires* (a) a scalar score, (b) a fenced editable surface, (c) a fixed iteration cost so experiments are commensurable.

Sources: github.com/karpathy/autoresearch.

### Pattern 4 — Reflect-and-Refine (verbal critique loop)

Self-Refine (Madaan et al., 2023): same LLM acts as generator → feedback → refiner; ~20% absolute gain across 7 tasks. Reflexion (Shinn et al., 2023) adds an **episodic memory buffer** of verbal reflections persisted across trials. Voyager generalizes with **GPT-4-as-critic** that reads `(state, task)` and returns pass/fail + critique. None publishes a hard convergence rule — they all use a max-iteration cap or a "user happy" signal.

Sources: arXiv 2303.17651 (Self-Refine), arXiv 2303.11366 (Reflexion), voyager.minedojo.org.

### Pattern 5 — Constitutional / Multi-Agent Critique (principled self-revision)

Anthropic's Constitutional AI: **generate → critique against a constitution → revise**, no human safety labels in the loop. Anthropic's `skill-creator` ships specialized subagents — `agents/grader.md`, `agents/comparator.md`, `agents/analyzer.md` — each a focused critic.

Sources: arXiv 2212.08073, anthropic.com/research/constitutional-ai-harmlessness-from-ai-feedback, github.com/anthropics/skills.

### Pattern 6 — Telemetry-Driven Evolution (production traces → re-tune)

Langfuse and Helicone capture every prod trace, attach LLM-as-judge + user-feedback scores, feed those into datasets that the next optimizer round consumes. Devin's autofix-on-review loop is the agentic version: any bot comment (linter, CI, security scanner) becomes an input to the agent, which patches the PR with no human in the loop for mechanical fixes.

Sources: langfuse.com, cognition.ai/blog/closing-the-agent-loop-devin-autofixes-review-comments.

## kiho mapping vs taxonomy

| Pattern | kiho coverage | Evidence |
|---|---|---|
| 1 Compile-via-Optimizer | absent | No skill named optimizer/teleprompter; 24-gate pipeline is rule-checking, not metric-search |
| 2 Scaffold-and-Fill | partial | `skill-create` implies a scaffold; gates suggest steps; no typed parameter schema or dry-run |
| 3 Generate-then-Verify-Loop | weak | Behavioral verification only at creation, not at update |
| 4 Reflect-and-Refine | absent | No critique agent; user is the reflector |
| 5 Constitutional / Multi-Agent Critique | partial | 24 gates ≈ a constitution but rule-checks not LM critiques. No grader/comparator/analyzer subagents |
| 6 Telemetry-Driven Evolution | partial | `evolution-scan` and `skill-learn` hint at it but the 9 observed gaps are all post-hoc human discoveries, not telemetry-fired |

**Strong**: rule-based gates exist (24-gate pipeline), specialized verbs (`derive`, `deprecate`, `find`, `learn`) for skill lifecycle, override mechanism (`soul-apply-override`).

**Absent**: (i) metric-driven optimization loop, (ii) typed parameter schema with dry-run, (iii) dedicated critic subagents per quality dimension, (iv) inbound-dependency graph as a gate, (v) cross-skill parity check, (vi) production-telemetry loop that fires regeneration without user prompting.

The diagnosis matches Deming exactly: kiho is doing inspection at the end rather than building quality in upstream.

## 7 missing pieces in `_meta` set

For each: **what's missing → primary-source pattern → kiho-fit constraint**.

### 1. `skill-spec` — typed parameter schema + dry-run preview

**Pattern**: Backstage `parameters:` JSONSchema + dry-run mode.
**Kiho-fit**: markdown-only ⇒ ship the schema as a YAML frontmatter block in each `_meta/` SKILL.md plus a fenced markdown table of "required artifacts produced." Dry-run = a pre-write step that emits a tree diff in the chat, not a tmp dir.

### 2. `skill-optimize` — metric-search loop on the trigger description

**Pattern**: Anthropic skill-creator's `scripts.run_loop --max-iterations 5` with 20 trigger-eval queries (mixed should-trigger / should-not-trigger), 60/40 train/test split, **best-by-test-score** selection.
**Kiho-fit**: no runtime DB ⇒ store eval queries as `skills/<name>/triggers.eval.md` (one fenced JSON block); loop is a sequence of Task-tool subagent calls; selection logged as a markdown checklist.

### 3. `skill-critic` — dedicated grader / comparator / analyzer subagents

**Pattern**: Anthropic skill-creator's `agents/grader.md`, `comparator.md`, `analyzer.md`; GEPA's reflective critic; Constitutional AI principle-checker.
**Kiho-fit**: depth-cap 3 ⇒ critic subagent must be exactly one level deep, invoked via Task with skill path + scorecard.md as inputs.

### 4. `skill-graph` — inbound dependency scan as a pre-gate

**Pattern**: Backstage `if:` step gating + custom field validators; Devin autofix architecture.
**Kiho-fit**: markdown-only ⇒ Grep-based scan over `skills/**/*.md` + 4 cross-anchor scan (CLAUDE.md, agents/*.md, README.md, templates/*.md) for stale references. Pre-regen, gate refuses if any inbound link would break.

### 5. `skill-parity` — cross-sibling structural diff

**Pattern**: GEPA Pareto frontier; Voyager's skill library indexed for similar-skill comparison.
**Kiho-fit**: no runtime DB ⇒ parity report as generated `_meta/parity.md` listing every sibling's directory layout side by side. Regeneration must declare a layout choice; gate refuses divergence without explicit `parity-exception:` frontmatter.

### 6. `skill-verify` — behavioral verification at every update, not just creation

**Pattern**: Karpathy autoresearch keep-or-discard gate; Devin 2.2 self-verify; Toyota jidoka stop-on-abnormality.
**Kiho-fit**: every `skill-improve` and `skill-derive` invocation runs the same verify steps as `skill-create`. Verify lives once in `_meta/skill-spec`, referenced by every other `_meta` skill (single source of truth, no copy-paste).

### 7. `skill-watch` — telemetry-driven regen trigger

**Pattern**: Langfuse/Langsmith production traces + LLM-as-judge feeding datasets that auto-trigger optimizer; Devin bot-comment-as-trigger; Reflexion episodic memory of failures.
**Kiho-fit**: CEO-only user interaction ⇒ `skill-watch` aggregates signals into a single ranked queue and presents the CEO with one **batch** decision per session, not per-skill prompts. Markdown-only ⇒ queue as generated `_meta/watch.md`.

## Target SOP

Modeled on a composition of (a) Anthropic skill-creator's iterative loop, (b) GEPA's Pareto frontier + textual reflection, (c) Backstage's typed-scaffold + dry-run, (d) Toyota jidoka stop-on-abnormality.

**Inputs (CEO provides once per batch)**:
1. A list of N skill specs as a markdown table: `name | one-line purpose | parent`.
2. A scorecard reference (`_meta/scorecard.md`).
3. A single "approve all" or "show me only the abnormal ones" decision at the end.

**Per-skill pipeline (deterministic, no human)**:

| Step | Pattern | Stops on |
|---|---|---|
| 1. Resolve `skill-spec` parameter struct | Backstage parameters | malformed input |
| 2. Inbound `skill-graph` scan | Devin pre-flight | broken refs that aren't auto-fixable |
| 3. Sibling `skill-parity` check | GEPA Pareto | unauthorized layout divergence |
| 4. Generate v1 (Reflect-and-Refine, max 3 rounds) | Self-Refine + Reflexion | round 3 |
| 5. `skill-critic` (grader + comparator + analyzer) | Constitutional AI critic | any layer scoring below threshold |
| 6. `skill-optimize` trigger-eval loop (max 5 iter, 20 queries, 60/40 split) | Anthropic skill-creator + DSPy MIPROv2 | test-score plateau or pass threshold |
| 7. `skill-verify` behavioral test (same suite that ran on creation) | Karpathy autoresearch keep/discard | val score worse than baseline |
| 8. Citation-verbatim check (Grep against source files) | Poka-yoke | any quote that isn't a substring of its citation |
| 9. Cousin-prompt robustness probe | GEPA Pareto across sibling triggers | drop in any sibling's eval score |
| 10. Stale-path scan across 4 known reference points | Poka-yoke | any unresolved path |

**Single CEO checkpoint** (per-batch, not per-skill):
- Generated `_meta/batch-report.md` lists per-skill: green (all gates passed), yellow (gates passed but critic score < 0.8 on some axis), red (jidoka-stopped, needs CEO call).
- CEO replies once with a bulk decision: "ship green, defer yellow, discuss red."

This matches Devin's stated reduction: *"the human's job narrows to decisions requiring judgment like architecture and product direction; everything mechanical gets caught and fixed before review."*

## 9-gap prevention table

| Gap | Pipeline prevention |
|---|---|
| Citation drift (paraphrased, not verbatim) | Step 8 — Grep-substring assertion: cited string MUST appear character-exact in named source; jidoka-stop on miss |
| Script-location inconsistency (`bin/` vs `scripts/`) | Step 3 — `skill-parity` refuses divergence without `parity-exception:` frontmatter; `skill-spec` declares canonical location once |
| Single-writer-pattern parity not auto-checked across siblings | Step 3 — sibling structural diff is mandatory; the pattern is encoded in `_meta/scorecard.md` as a poka-yoke rule |
| Behavioral verification only at creation, not at update | Step 7 — `skill-verify` runs on every `_meta/skill-improve` and `_meta/skill-derive`, sourced from the same `_meta/skill-spec` block |
| Quality-scorecard 7-layer not auto-applied | Step 5 — `skill-critic` invokes a subagent per layer (grader, comparator, analyzer mirror Anthropic's `agents/*.md`) |
| Inbound dependency scan not auto-run pre-regen | Step 2 — `skill-graph` runs before any write; scaffolder pattern from Backstage forbids step-N if step-2 fails |
| Stale path references across 4 files not detected | Step 10 — Grep-based scan of the 4 known reference points; jidoka-stop on any unresolved path |
| Cousin-prompt robustness never measured | Step 9 — re-run sibling skills' trigger-evals after this skill changes; if any drops, GEPA-style retain prior Pareto candidate |
| Trigger-eval loop never executed | Step 6 — `skill-optimize` is non-optional; the loop is the same `scripts.run_loop` shape Anthropic's skill-creator uses |

**One-line summary** (Shingo): every gap above is a *mistake* that became a *defect* because the corresponding poka-yoke wasn't installed in the pipeline. The seven new `_meta` skills install seven poka-yokes; together they make the nine specific gaps structurally impossible.
